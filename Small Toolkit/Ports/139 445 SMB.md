When SMB is found conduct the following enumeration steps:

- Enumerate Host
    - `crackmapexec smb [ip]`
- List Shares
    - `crackmapexec smb [host/ip] -u [user] -p [pass] --shares`
    - `crackmapexec smb [host/ip] -u guest -p '' --shares`
    - `smbclient -N -L //[ip]`
- Logging in and Accessing Shares
    - `smbclient //[host]/[sharename] -U '[username]%[password]'`
- Enumerate Files
    - `smbclient //[ip]/[share] -N`
    - `smbclient //[ip]/[share] -U [username] [password]`
    - `crackmapexec smb -u [user] -p [pass] -M spider_plus`
    - `smbclient.py '[domain]/[user]:[pass]@[ip/host] -k -no-pass` - Kerberos auth
    - `manspider.py --threads 256 [IP/CIDR] -u [username] -p [pass] [options]`
- User enumeration
    - RID Cycling
        - `lookupsid.py guest@[ip] -no-pass`
        - `crackmapexec smb [ip] -u guest -p '' --rid-brute`
    - SAM Remote Protocol - `samrdump.py [domain]/[user]:[pass]@[ip]`
- Check for Vulnerabilities - `nmap --script smb-vuln* -p 139,445 [ip]`

The command [net use](https://docs.microsoft.com/en-us/previous-versions/windows/it-pro/windows-server-2012-r2-and-2012/gg651155\(v=ws.11\)) connects a computer to or disconnects a computer from a shared resource or displays information about computer connections. find how many files the shared folder and its subdirectories contain.

```
dir n: /a-d /s /b | find /c ":\"
```

| `dir`  | Application                                                    |
| ------ | -------------------------------------------------------------- |
| `n:`   | Directory or drive to search                                   |
| `/a-d` | `/a` is the attribute and `-d` means not directories           |
| `/s`   | Displays files in a specified directory and all subdirectories |
| `/b`   | Uses bare format (no heading information or summary)           |
Search for specific names in files

```
n:\*[name]* /s /b
```

Result : 
n:\Contracts\private\ [name].txt

search for a specific word within a text file, we can use [findstr](https://docs.microsoft.com/en-us/windows-server/administration/windows-commands/findstr).

```
findstr /s /i cred n:\*.*
```

Result : 
n:\Contracts\private\secret.txt:file with all credentials

#### Linux

Linux (UNIX) machines can also be used to browse and mount SMB shares. Note that this can be done whether the target server is a Windows machine or a Samba server. Even though some Linux distributions support a GUI, we will focus on Linux command-line utilities and tools to interact with SMB. Let's cover how to mount SMB shares to interact with directories and files locally.

#### Linux - Mount

  Interacting with Common Services

```shell-session
Retric@htb[/htb]$ sudo mkdir /mnt/Finance
Retric@htb[/htb]$ sudo mount -t cifs -o username=plaintext,password=Password123,domain=. //192.168.220.129/Finance /mnt/Finance
```

As an alternative, we can use a credential file.

  Interacting with Common Services

```shell-session
Retric@htb[/htb]$ mount -t cifs //192.168.220.129/Finance /mnt/Finance -o credentials=/path/credentialfile
```

The file `credentialfile` has to be structured like this:

#### CredentialFile

Code: txt

```txt
username=plaintext
password=Password123
domain=.
```

Note: We need to install `cifs-utils` to connect to an SMB share folder. To install it we can execute from the command line `sudo apt install cifs-utils`.

Once a shared folder is mounted, you can use common Linux tools such as `find` or `grep` to interact with the file structure. Let's hunt for a filename that contains the string `cred`:

#### Linux - Find

  Interacting with Common Services

```shell-session
Retric@htb[/htb]$ find /mnt/Finance/ -name *cred*

/mnt/Finance/Contracts/private/credentials.txt
```

Next, let's find files that contain the string `cred`:

  Interacting with Common Services

```shell-session
Retric@htb[/htb]$ grep -rn /mnt/Finance/ -ie cred

/mnt/Finance/Contracts/private/credentials.txt:1:admin:SecureCredentials!
/mnt/Finance/Contracts/private/secret.txt:1:file with all credentials
```
