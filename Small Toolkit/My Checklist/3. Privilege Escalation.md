# 🎯 What You’re Looking For

## 1️⃣ Credential Leakage
Examples:

DB_PASSWORD=SuperSecret123  
AWS_SECRET_ACCESS_KEY=...  
JWT_SECRET=...

Very common in:

- Docker containers
- Node apps
- Python Flask apps
- Misconfigured systemd services

---

## 2️⃣ PATH Manipulation Opportunities

If:

PATH=/home/user/bin:/usr/local/bin:/usr/bin

Writable directories in PATH = possible privilege escalation vector.

---

## 3️⃣ Application Secrets

Metabase example (since you were working on it earlier):

Environment variables sometimes store:

- MB_DB_CONNECTION_URI
- MB_ENCRYPTION_SECRET_KEY
- DATABASE_URL

That can lead to database access → credential reuse → lateral movement.
